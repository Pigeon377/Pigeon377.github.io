快速幂

这几天学了下矩阵快速幂 就顺便放一起写篇博客

普通的求幂算法

普通求幂咱就不说了，按照朴素的写法，求 $x{^y}$ ,将y个x直接相乘，
即使用循环一个一个乘，时间复杂度为O(n)

看的出来，写起来挺简单，而且不容易出错，但是时间复杂度太高了啦！

快速幂算法

由此，我们整了点新活，能快比朴素的求幂算法快上不少.
即通过对二进制的巧妙利用，能够将时间复杂度降低到 $O(log{_2}n)$ 的水准

举个栗子:

假设我们要求 $x{^{27}}$
又因为17的二进制码为 0001 1011
即
$$17 = 2{^4} + 2{^3} + 2{^1} + 2{^0}$$
那么， 可以得到
$$x{^{27}} = x{^{2{^4}}} + x{^{2{^3}}} + x{^{2{^1}}} + x{^{2{^0}}}$$
又因为
$$x{^{2{^1}}} = x{^{2{^0}}} *x{^{2{^0}}}$$
$$x{^{2{^4}}} = x{^{2{^3}}} *x{^{2{^3}}}$$
不难看出，如果构造一个一直保持倍增的乘数，可以在$O(log{_2}n)$时间下拿到正确答案

矩阵快速幂

其实矩阵快速幂和上面快速幂是差不多的啦，
不过是把对象从数换成了矩阵而已啦，
人家直接贴代码好了。

应用

话说，其实矩阵快速幂，好像在求斐波那契数列上更有帮助呐

假设 我们要求斐波那契数列的第N项，除了递推 $O(n)$ 和递归 $O(n{^n})$ 外，
还想整点花 $O(log{n})$ 的，可以这么做
