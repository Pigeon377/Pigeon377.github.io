Scala学习笔记

2021年11月14日

嗨呀！~ 这门语言是真的复杂

你看看隔壁Kotlin!

这几天在啃<<Scala函数式编程>>,说真的，我觉得里面写的函数式数据结构的内容,

真的非常反人类,不过学就学了,稍微记下笔记？

  • 首先，scala中没有static关键字，取而代之的是object，一个单例对象，
    单例对象就不说了，想不起来就爪巴回去读设计模式。单例对象不可通过new实例化，
    单例对象中的所有方法都近似等价于static修饰的方法,需要类名才能访问，老实说，
    有点像(其实就是)把所有的静态方法和静态字段集合起来。并且object类不能提供
    构造器参数，也就是说object必须是无参的。这个特殊的对象的构造器在此对象第一次
    被调用的时候才会真正被使用。同时，我们还可以定义一个与object相同的class，一般
    称该object为该class的伴生对象，二者必须定义在同一个文件中，二者可以互相访问彼此的私有成员
  • 函数式似乎强调数据共享？例如，给你一个List，让你删除第一个元素，你可以返回
    List的第二个节点作为新的List的开头，这样既没有改变原有的List，避免了开销，又达成了目标.
    (毕竟函数式编程对mutable和side effects深恶痛绝，可以保证List不会因为后续的操作而改变，如果实在要变，
    那为什么我们不返回一个新的List呢，都用scala了还在乎性能？(^▽^))
  • Scala的数据结构，大都持有header和tail两个字段，前者返回第一个元素，后者返回除了第一个元素
    的其余所有元素，只能说利好递归思想，可以，这很函数式~。
  • 放置在object中的apply方法,会在我们直接使用val x = object_name(params:???)的时候调用，
    等价于val x = object_name.apply(params:???)，算是个挺甜的语法糖。补一下，这里看到一个说法，
    apply是类伴生对象的工厂方法，好像挺对的,我们可以通过调用apply产生新的对象。
  • Scala中没有运算符的概念，所有运算符都是方法，就C++那种感觉，爽!
  • 对一个函数，val t = (x:Int, y:Int) => x+y (只是举个栗子),我们可以这样写，
    val a = t(114514, _),这样,a会得到一个带一个参数的Lambda表达式，假如调用
    a(1805296),就会得到1919810，其实就是大家最爱的curry，确实挺方便的，当然你写成
    def t(x:Int)(y:Int) = x+y，但我还是觉得前面那种更爽，至少现在是这样。
  • Scala编写Future / Option时,报错 No implicit found for parameter executor: ExecutionContext。
    找不到指定的隐式转换。查了几天资料，CSDN这种垃圾不是机翻就是特么的抄袭，越做越垃圾只能说是这群人活该。
    最后在StackOverFlow上面翻到答案，使用import scala.concurrent.ExecutionContext.Implicits.global引入隐式转换即可.
    要是早点用google能浪费这么久啊?😅

哎呀，您都读到这里了，我只能说，看心情再写呗，以上。
